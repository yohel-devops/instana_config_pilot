## 📋 Pasos para Levantar el Proyecto

### 1. Levantar con Docker (Recomendado)

```bash
# Desde la raíz del proyecto
docker-compose up --build
```

Esto levantará:
- **Backend**: http://localhost:8000
- **Frontend**: http://localhost:80

### 2. Verificar que el Backend está Corriendo

Abrir en el navegador:
- API Docs: http://localhost:8000/api/docs
- Health Check: http://localhost:8000/health

### 3. Usar la Aplicación

1. Abrir: http://localhost:80
2. La app cargará automáticamente 183 sensores del configuration.yaml
3. Buscar y seleccionar sensores (ej: "db2")
4. Click "Add" para habilitar
5. Click "Edit" para configurar
6. Ver preview del YAML generado con nombres correctos
7. Descargar configuration.yaml

El backend genera YAML con el formato correcto de Instana:

```yaml

# Source: Default Instana template v2026.05
# Template: v2026.05

com.instana.plugin.host:
  enabled: true
  tags:
    - client=IBM
    - environment=PROD
    - zone=test
    - owner=admin

com.instana.plugin.db2:
  enabled: true
  local:
    - instance: 'db2inst1'
      port: '50000'
      user: 'db2inst1'
      password: 'password'
      poll_rate: 5
      databases:
        - 'database1'
```

## 🐛 Troubleshooting

### Error: "Backend not available"
```bash
# Verificar que Docker esté corriendo
docker ps

# Reiniciar contenedores
docker-compose down
docker-compose up --build
```

### Error: "Port already in use"
```bash
# Cambiar puertos en docker-compose.yml
# Backend: 8000 -> 8001
# Frontend: 80 -> 8080
```

### Frontend no carga sensores
1. Verificar backend: http://localhost:8000/health
2. Verificar API docs: http://localhost:8000/api/docs
3. Revisar logs: `docker-compose logs backend`
